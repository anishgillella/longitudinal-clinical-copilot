2. **Edit signals** - Modify intensity, domain mapping, or evidence
3. **Add signals** - Manually add observations the AI missed
4. **Add notes** - Clinical observations not from transcript
5. **View transcript** - Jump to specific moments in conversation

## Session Types

| Type | Duration | Purpose | When to Use |
|------|----------|---------|-------------|
| **Intake** | 45-60 min | Initial comprehensive assessment | First session with new patient |
| **Check-in** | 15-20 min | Regular progress monitoring | Ongoing sessions |
| **Targeted Probe** | 20-30 min | Deep dive into specific domain | When domain needs exploration |
| **Follow-up** | 10-15 min | Quick status check | Between major sessions |

## Best Practices

### Before Session
1. Review patient's domain scores and trends
2. Check AI recommendations for focus areas
3. Note any concerns from previous sessions
4. Fill out pre-session check-in

### During Session
1. Let AI agent lead the conversation
2. Take notes on observations not captured in audio
3. Flag any safety concerns immediately

### After Session
1. Review all extracted signals carefully
2. Confirm or reject each AI observation
3. Add any signals the AI missed
4. Write brief clinician notes
5. Check hypothesis changes make sense

### Longitudinal
1. Review hypothesis trends monthly
2. Identify patterns across sessions
3. Adjust focus areas based on data gaps
4. Generate reports at milestone points
